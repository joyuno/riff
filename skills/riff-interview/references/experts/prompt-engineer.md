## Q1: 결정사항 충돌 감지

모든 전문가 결정사항을 읽고 충돌 항목 탐지:

```
충돌 감지 패턴:
- 프론트엔드 기술 결정 ↔ 백엔드 기술 결정 불일치
- 데이터 엔지니어 인프라 ↔ DB 설계자 스택 불일치
- AI 엔지니어 LLM 비용 ↔ 데이터 엔지니어 인프라 예산 불일치
```

충돌 발견 시:
"[충돌 내용]이 발견됐습니다. 어떤 결정을 우선할까요?"
- A: [옵션 1]
- B: [옵션 2]

충돌 없으면 → Q2로 바로 이동

## 에이전트별 master-plan.md 슬라이스 기준

master-plan.md 전체를 모든 에이전트에게 전달하면 토큰 낭비다.
각 에이전트에게는 필요한 섹션만 슬라이스해서 전달한다.

| 에이전트 | 전달 섹션 | 제외 섹션 |
|---------|----------|----------|
| frontend-agent | UI/UX + 프론트엔드 + P0/P1 | 백엔드, DB, 인프라, AI |
| backend-agent | 백엔드 + DB + 인증 + P0/P1 | UI/UX, 프론트엔드 |
| ai-agent | AI + 백엔드(API 부분만) + P0/P1 | UI/UX, 프론트엔드, DB |
| infra-agent | 데이터/인프라 + DB + P0/P1 | UI/UX, 프론트엔드, AI |

모든 에이전트에게 공통 전달:
- "프로젝트 요약" 섹션 (2-3문단)
- "제약 조건" 섹션
- Architecture Contract 경로

---

## 토큰 최적화 규칙

최종 기획서 작성 시:
1. 각 에이전트 프롬프트에 전달할 섹션만 포함 (전체 문서를 모든 에이전트에 전달 금지)
2. 에이전트별 필요 섹션만 슬라이스:
   - frontend-agent → UI/UX + 프론트엔드 섹션만
   - backend-agent → 백엔드 + DB + 인증 섹션만
   - ai-agent → AI 섹션만
3. 마스터 기획서 전체는 오케스트레이터만 보유
