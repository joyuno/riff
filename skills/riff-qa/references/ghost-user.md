# 유령 사용자 QA — 상세 가이드

유령 사용자(Ghost User)는 AI가 스크린샷을 보며 목표를 향해 자율적으로 앱을 탐색하는 QA 변형이다.
사전에 작성된 시나리오를 따르지 않고, 실제 사용자처럼 UI를 보고 스스로 경로를 찾는다.
직관적이지 않은 UI, 막히는 지점, UX 이슈를 발견하는 데 특화되어 있다.

---

## 핵심 특성

| 일반 Tier 3 시나리오 | 유령 사용자 |
|---------------------|-----------|
| 정해진 Step을 순서대로 실행 | 목표만 주어지고 경로는 자율 결정 |
| 예상 결과가 미리 정의됨 | 막히면 다른 경로 시도 |
| 기술적 버그 탐지 중심 | UX 이슈, 직관성 문제 탐지 중심 |
| 재현 가능한 스크립트 | 탐색 과정 자체가 증거 |

---

## 실행 방법

### 에이전트 프롬프트 템플릿

```
당신은 [앱 이름]을 처음 사용하는 일반 사용자입니다.
기술적 배경지식은 없으며, 화면에 보이는 것만으로 판단합니다.

목표: [구체적인 목표]
예시: "새 주문을 생성하고 결제까지 완료하세요"
예시: "내 프로필 정보를 수정하세요"
예시: "지난달 매출 보고서를 다운로드하세요"

규칙:
1. 스크린샷을 보고 다음 행동을 결정하세요
2. 막히면 최대 3번까지 다른 방법을 시도하세요
3. 3번 시도 후에도 안 되면 "막힘" 으로 기록하고 다음 목표로 이동하세요
4. 각 행동마다 스크린샷을 찍으세요
5. 헷갈리거나 모호한 UI 요소는 메모하세요

시작 URL: http://localhost:3000
```

### 탐색 루프

```
browser_navigate(시작 URL)
browser_take_screenshot()  → 첫 화면 확인

루프:
  browser_snapshot()          → 현재 요소 파악
  [AI가 목표 기반으로 다음 행동 결정]
  browser_click() 또는 browser_fill_form()
  browser_take_screenshot()   → 행동 결과 캡처
  browser_console_messages()  → 에러 확인

  목표 달성 여부 판단:
    → 달성: 완료 기록
    → 미달성: 다른 경로 시도 또는 막힘 기록
```

## 발견 목표별 탐색 전략

### 1. 핵심 전환 흐름

목표: "서비스의 핵심 가치를 달성하는 경로가 직관적인가"

```
예시 목표들:
- 회원가입 후 첫 구매까지
- 로그인 후 핵심 기능 실행까지
- 설정 변경 후 효과 확인까지
```

기록 항목:
- 목표 달성까지 걸린 클릭 수
- 예상과 다른 경로로 우회한 횟수
- 막힌 지점과 이유

### 2. 빈 상태(Empty State) 탐색

목표: "데이터가 없는 초기 상태에서 UI가 적절히 안내하는가"

```
탐색 방법:
- 새 계정으로 로그인
- 데이터가 없는 페이지 방문
- 빈 목록, 빈 대시보드 확인
```

확인 항목:
- 빈 상태 메시지가 존재하는가
- 다음 행동으로 안내하는 CTA가 있는가
- 빈 상태에서 에러가 발생하는가

### 3. 에러 복구 흐름

목표: "에러가 발생했을 때 사용자가 복구할 수 있는가"

```
탐색 방법:
- 잘못된 데이터 입력 후 제출
- 폼 오류 발생 시 메시지 확인
- 에러 후 정상 입력으로 재시도
```

확인 항목:
- 에러 메시지가 명확하고 도움이 되는가
- 에러 후 이전 입력값이 유지되는가
- 에러 복구 경로가 명확한가

## 기록 대상

### UX 이슈 유형

| 유형 | 예시 | 심각도 |
|------|------|--------|
| 찾기 어려운 버튼 | "저장" 버튼이 화면 하단 스크롤 뒤에 숨겨짐 | Medium |
| 모호한 레이블 | "관리" 버튼이 무엇을 관리하는지 불명확 | Low |
| 막힌 경로 | 목표를 달성할 방법을 3번 시도 후 발견 못 함 | High |
| 피드백 없는 행동 | 버튼 클릭 후 아무 반응 없음 | High |
| 예상과 다른 결과 | "삭제" 클릭 시 확인 없이 즉시 삭제 | Medium |
| 언어 불일치 | UI는 한국어인데 에러 메시지가 영어 | Low |

### 접근성 이슈

```bash
# 접근성 스냅샷에서 확인할 항목
browser_snapshot() 결과에서:
- img 요소에 alt 텍스트 없음
- button에 name/label 없음 (스크린 리더 미지원)
- 키보드로 접근 불가능한 상호작용 요소
```

### i18n 이슈

- 에러 메시지 언어가 UI 언어와 불일치
- 하드코딩된 날짜/통화 형식
- 번역되지 않은 플레이스홀더 텍스트

## 유령 사용자 보고서 형식

### 탐색 로그

```
목표: 새 주문 생성

시도 1:
  행동: 대시보드에서 "주문" 메뉴 클릭
  결과: 주문 목록 페이지로 이동 (새 주문 버튼 없음)
  스크린샷: ghost-01-order-list.png

시도 2:
  행동: 상단 네비게이션의 "+" 아이콘 클릭
  결과: 알림 추가 팝업 열림 (예상과 다름)
  스크린샷: ghost-02-wrong-popup.png

시도 3:
  행동: 사이드바 하단 "새로 만들기" 버튼 클릭
  결과: 주문 생성 폼 표시됨
  스크린샷: ghost-03-create-form.png

결과: 달성 (3번 시도)
UX 이슈: 주문 생성 경로가 직관적이지 않음. "새로 만들기" 레이블이 모호함.
```

### 발견 요약

```
탐색 목표 수: 5개
달성: 3개 (1회 시도: 1개, 2회 시도: 1개, 3회 시도: 1개)
미달성 (막힘): 2개

발견된 UX 이슈:
  [HIGH]   보고서 다운로드 경로를 3번 시도 후 미발견
  [MEDIUM] 주문 생성 버튼 위치 직관성 부족 (3번 시도 필요)
  [LOW]    에러 메시지가 영어로 표시됨 (i18n 이슈)

접근성 이슈:
  - 상품 이미지 alt 텍스트 없음 (3개 요소)
  - "편집" 아이콘 버튼에 label 없음
```
