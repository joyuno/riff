### VC-Q1: 영상 유형 및 출력 플랫폼

**질문:**
> "어떤 종류의 영상을 만들어야 하나요? 유튜브 롱폼(10분+), 유튜브 쇼츠/릴스/틱톡(1분 이하), 사내 교육 영상, 마케팅 광고 영상 중 어느 것인가요? 그리고 동시에 여러 플랫폼에 올려야 하나요?"

**왜 이 질문이 중요한가:**
영상 유형이 종횡비(16:9 vs 9:16), 길이, 자막 위치, 편집 스타일을 결정합니다. 플랫폼별 포맷이 다르기 때문에 처음부터 멀티포맷 지원 여부를 결정해야 합니다.

**관련 Layer:** WHAT, HOW

**답변 분기:**

| 영상 유형 | 포맷 | 핵심 요구사항 |
|---|---|---|
| 유튜브 롱폼 | 16:9, 10분+ | 챕터 마커, 긴 대본 처리, 섬네일 A/B 테스트 |
| 쇼츠/릴스/틱톡 | 9:16, 60초 이하 | 훅 도입부 강조, 자막 자동 삽입, 빠른 컷 |
| 교육 영상 | 16:9, 다양한 길이 | 자막 정확도 높음, 챕터 구조, 퀴즈 연동 가능성 |
| 마케팅 광고 | 1:1 또는 16:9, 15-60초 | CTA 삽입, 브랜드 가이드라인 준수 |
| 멀티포맷 | 복수 | 마스터 편집 후 플랫폼별 리사이즈 자동화 |

### VC-Q2: 제작 파이프라인 범위

**질문:**
> "영상 제작 과정에서 어느 단계부터 어느 단계까지 자동화하고 싶으신가요? 기획→대본→음성→영상 편집→자막→썸네일→업로드 중에서 현재 가장 오래 걸리는 단계는 어디인가요?"

**왜 이 질문이 중요한가:**
파이프라인 전체 자동화와 특정 단계 자동화는 복잡도가 10배 이상 차이납니다. 현재 병목 단계 하나를 해결하는 것이 MVP입니다.

**관련 Layer:** WHY, WHAT

**단계별 자동화 난이도:**

| 단계 | 자동화 도구 | 난이도 | 품질 |
|---|---|---|---|
| 기획/아이디어 | LLM (GPT/Claude) | 낮음 | 보조적 |
| 대본 생성 | LLM + 프롬프트 | 낮음 | 중간 (검토 필요) |
| 음성 합성 (TTS) | ElevenLabs / OpenAI TTS | 낮음 | 높음 |
| 영상 편집 | Remotion / FFmpeg | 높음 | 중간 |
| AI 영상 생성 | Runway / Sora / Kling | 높음 | 낮음~중간 |
| 자막 생성 | Whisper (OpenAI) | 낮음 | 높음 |
| 썸네일 생성 | DALL-E / Midjourney + 합성 | 중간 | 중간 |
| 자동 업로드 | YouTube Data API | 낮음 | 높음 |

### VC-Q3: AI 활용 범위

**질문:**
> "AI가 영상 제작의 어느 부분을 담당하면 좋겠나요? 대본만 AI가 쓰고 나머지는 직접 편집하는 건가요, 아니면 완전히 사람 없이 영상이 자동으로 만들어지는 풀 자동화를 원하시나요?"

**왜 이 질문이 중요한가:**
"완전 자동화"는 현재 AI 기술 수준에서 품질 타협이 불가피합니다. 어느 단계에서 사람이 개입할지 결정해야 현실적인 시스템을 설계할 수 있습니다.

**관련 Layer:** HOW

**답변 분기:**

| 자동화 수준 | 설명 | 품질 vs 속도 |
|---|---|---|
| 보조 자동화 | AI가 대본 초안 작성, 사람이 검토 후 편집 | 품질 높음, 시간 50% 절감 |
| 반자동화 | 대본+음성 자동, 편집 템플릿 적용, 사람이 최종 검토 | 중간 품질, 시간 80% 절감 |
| 완전 자동화 | 기획부터 업로드까지 무인 | 품질 타협, 시간 95% 절감 |

**완전 자동화 경고:**
> 완전 자동화 영상은 플랫폼 알고리즘에서 저평가될 수 있습니다. YouTube는 반복적인 AI 생성 콘텐츠를 감지하고 노출을 줄이는 것으로 알려져 있습니다. 최소한 썸네일과 제목에는 사람의 판단이 개입하는 것을 권장합니다.

### VC-Q4: 음성 및 언어 요구사항

**질문:**
> "영상의 음성은 어떻게 처리할 건가요? 직접 녹음한 목소리를 사용하나요, AI 음성(TTS)을 쓰나요? 그리고 한국어만인가요, 다국어 버전이 필요한가요?"

**왜 이 질문이 중요한가:**
TTS 품질과 언어 지원이 도구 선택을 결정합니다. 한국어 TTS는 영어보다 선택지가 적고, 다국어는 자막 번역 파이프라인이 추가됩니다.

**관련 Layer:** HOW

**답변 분기:**

| 음성 방식 | 추천 도구 | 월 비용 기준 |
|---|---|---|
| AI 음성 (한국어) | ElevenLabs, ClovaVoice (CLOVA) | $5~$99/월 |
| AI 음성 (영어) | ElevenLabs, OpenAI TTS, Azure TTS | $5~$50/월 |
| 내 목소리 클론 | ElevenLabs Voice Clone | $22/월~ |
| 다국어 자동 번역 | DeepL API + TTS | 언어당 추가 비용 |
| 자막만 (음성 없음) | — | 무료 (Whisper 자체 호스팅) |

### VC-Q5: 브랜드 가이드라인 적용

**질문:**
> "영상마다 동일하게 적용되어야 하는 브랜드 요소가 있나요? 로고 위치, 폰트, 색상, 인트로/아웃트로 영상, 자막 스타일 같은 것들이요. 영상 100개 이상에 일관성이 필요한가요?"

**왜 이 질문이 중요한가:**
브랜드 일관성 요구가 있으면 영상 편집 템플릿 시스템이 필요합니다. Remotion 또는 FFmpeg 필터 체인으로 구현하며, 설계 초기부터 고려해야 합니다.

**관련 Layer:** WHAT, HOW

**답변 분기:**

| 브랜드 요구 | 구현 방식 |
|---|---|
| 없음 (자유롭게) | 도구 제약 없음, 빠른 구현 가능 |
| 인트로/아웃트로만 | FFmpeg concat + 고정 영상 파일 |
| 자막 스타일 통일 | 자막 CSS/스타일 설정 파일 중앙 관리 |
| 완전한 브랜드 시스템 | Remotion 템플릿 컴포넌트 개발 |

### VC-Q6: 영상 길이와 처리 시간 허용 범위

**질문:**
> "영상 하나를 만드는 데 얼마나 걸려도 괜찮은가요? 실시간으로 즉시 만들어져야 하나요, 몇 시간 후에 완성되어도 괜찮은가요? 그리고 하루에 몇 개 영상을 만들어야 하나요?"

**왜 이 질문이 중요한가:**
처리 시간 허용 범위가 인프라 비용을 결정합니다. 즉시 생성은 GPU 서버가 필요하고, 배치 처리는 훨씬 저렴합니다.

**관련 Layer:** HOW, MEASURE

**답변 분기:**

| 처리 시간 | 인프라 | 비용 |
|---|---|---|
| 즉시 (실시간, 수분 이내) | GPU 서버 또는 API (Runway 등) | 높음 ($$$) |
| 30분~2시간 | 비동기 큐 + 일반 서버 | 중간 ($$) |
| 수 시간 (배치) | 배치 처리 + 저렴한 서버 | 낮음 ($) |
| 하루 1~5개 | 단일 서버로 충분 | 최소 |
| 하루 50개 이상 | 병렬 처리 큐 필요 | 중간 이상 |

### VC-Q7: 업로드 자동화 및 메타데이터 관리

**질문:**
> "영상 업로드까지 자동화하고 싶으신가요? 그렇다면 제목, 설명, 태그, 카테고리, 공개 시간 예약까지 자동으로 설정해야 하나요? 각 플랫폼의 API 사용 조건을 이미 확인하셨나요?"

**왜 이 질문이 중요한가:**
YouTube API는 무료 쿼터 제한이 있고, 자동 업로드는 정책 위반 위험이 있습니다. 메타데이터(제목/설명) 최적화는 별도의 SEO 로직이 필요합니다.

**관련 Layer:** WHAT

**플랫폼별 자동화 가능 여부:**

| 플랫폼 | API | 쿼터/제한 | 주의사항 |
|---|---|---|---|
| YouTube | YouTube Data API v3 | 일 10,000 단위 | 과도한 자동화 시 채널 제재 위험 |
| Instagram / Reels | Meta Graph API | 사용량 제한 있음 | 비즈니스 계정 필요 |
| TikTok | TikTok for Business API | 승인 필요 | 개인 계정 자동화 불가 |
| Naver TV | 공식 업로드 API 없음 | — | 수동 업로드만 가능 |
