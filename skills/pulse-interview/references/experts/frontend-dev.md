# 전문가: 프론트엔드 개발자

**기여 Layer**: Layer 2 (WHO) + Layer 3 (WHAT) + Layer 4 HOW-C
**상세 질문 흐름**: references/enriched-layers.md 참조
**이 파일**: Layer 4 HOW-C 질문의 전체 선택지와 화면 구성 로직 상세

**역할**: 페이지 구성, 기능 상세, 기술 스택 확정
**질문 수**: 5개
**의존**: UI 디자이너 결정사항 + Layer 3 WHAT 완료 후 실행

---

## 도입 멘트

"저는 프론트엔드 개발자입니다. 화면 구성과 기능을 구체화해볼게요."

---

## Q1: 페이지 구성

"이 서비스의 주요 화면을 나열해볼게요. 제가 먼저 MVP에 필요할 것 같은 페이지를 추천드릴게요."

Layer 3 WHAT 기반으로 자동 추천:
```
추천 페이지 목록:
1. 랜딩/홈 — 서비스 소개, CTA
2. 로그인/회원가입 — 인증 플로우
3. [핵심 기능 페이지 1] — [설명]
4. [핵심 기능 페이지 2] — [설명]
5. 설정/프로필 — 사용자 관리

추가할 페이지가 있나요? 제거할 페이지가 있나요?"
```

---

## Q2: 페이지별 핵심 기능 상세

각 확정된 페이지에 대해 순서대로:
"[페이지명]에서 사용자가 할 수 있어야 하는 것을 모두 나열해주세요."

예시 (홈 페이지):
- 검색 기능 필요한가요? (실시간 vs 버튼 클릭)
- 필터/정렬 필요한가요?
- 무한 스크롤 vs 페이지네이션?
- 빈 상태(데이터 없을 때) 화면이 필요한가요?

---

## Q3: 라우팅 / 네비게이션

"사용자가 페이지 간 이동하는 방식은?"

선택지:
- A: 상단 네비게이션 바 (웹 표준)
- B: 사이드바 (대시보드, 관리자)
- C: 하단 탭 바 (모바일 앱 스타일)
- D: 계층형 드릴다운 (카테고리 → 상세)
- E: 조합 (설명해주세요)

---

## Q4: 상태 관리 복잡도

"화면 간 공유해야 하는 데이터가 있나요?"

예시 질문:
- 로그인 상태를 모든 페이지에서 알아야 하나요? (거의 항상 yes)
- 장바구니처럼 여러 페이지에서 공유하는 데이터가 있나요?
- 실시간으로 업데이트되어야 하는 데이터가 있나요? (채팅, 알림 등)

태깅: `state_simple`(로컬만) / `state_shared`(전역) / `state_realtime`(실시간)

---

## Q5: SEO / PWA 요구사항

"다음 중 필요한 것을 선택해주세요."
- [ ] 검색엔진 노출 필요 (SEO → Next.js SSR/SSG 필요)
- [ ] 앱처럼 설치 가능해야 함 (PWA)
- [ ] 오프라인 동작 필요
- [ ] 해당 없음 (내부 도구, 로그인 후에만 사용)
