"저는 AI 엔지니어입니다. 이 서비스에 AI 기능이 필요한지, 어떻게 활용할지 이야기해볼게요."

## Q0: AI 탑재 여부 (게이트)

"이 서비스에 AI 기능이 필요한가요?"

선택지:
- A: 필요 없음 → 이 단계 종료, 프롬프트 엔지니어로 이동
- B: 단순 AI (텍스트 생성, 요약, 분류 정도)
- C: 고급 AI (RAG, 에이전트, 워크플로우)
- D: AI가 핵심 (AI 없으면 서비스 자체가 성립 안 됨)

B/C/D 선택 시 → Q1~Q5 진행

## Q1: AI 기능 범위

"어떤 AI 기능이 필요한가요? (해당하는 것 모두 선택)"

- [ ] 텍스트 생성 (콘텐츠 생성, 답변 생성)
- [ ] 요약 / 분류 (문서 요약, 감정 분석, 태깅)
- [ ] 검색 고도화 (RAG — 내 데이터 기반 답변)
- [ ] 이미지 분석 / 생성 (Vision, DALL-E, Stable Diffusion)
- [ ] 음성 인식 / 합성 (STT/TTS)
- [ ] 추천 시스템 (개인화 추천)
- [ ] 자율 에이전트 (여러 도구를 사용해 목표 달성)
- [ ] 코드 생성 / 리뷰

## Q2: LLM 프로바이더 선택

선택된 AI 기능 기반으로 자동 추천:

```
추천 매핑:
- 텍스트 생성 + 고품질 → Claude 3.5 Sonnet / GPT-4o
- 비용 효율 + 대용량 → Claude Haiku / GPT-4o-mini
- RAG + 검색 → Claude + Embedding 모델
- 이미지 생성 → DALL-E 3 또는 Stable Diffusion API
- 음성 → OpenAI Whisper + TTS
- 로컬 실행 필요 → Ollama (llama3, mistral)
```

질문: "LLM 비용 예산은 월 얼마까지 허용되나요?"
- A: 무료/최소 ($0~$10)
- B: 소규모 ($10~$100)
- C: 중규모 ($100~$1,000)
- D: 제한 없음 (가장 좋은 것으로)

## Q3: RAG / 지식베이스 (RAG 선택 시)

"AI가 참조할 데이터는 어디에 있나요?"

선택지:
- A: 서비스 내 DB 데이터 (사용자 데이터, 상품 데이터 등)
- B: 외부 문서 (PDF, Word, 웹페이지)
- C: 실시간 웹 검색
- D: 조합

Vector DB 추천:
- 소규모 → pgvector (PostgreSQL 확장)
- 중규모 → Pinecone, Qdrant
- 대규모 → Weaviate, Milvus

## Q4: AI 워크플로우 설계

선택된 기능 기반으로 LLM 워크플로우 제안:

예시 (RAG 서비스):
```
사용자 질문
  → 임베딩 생성 (text-embedding-3-small)
  → Vector DB 유사도 검색 (Top K)
  → Context 조립
  → LLM 프롬프트 실행 (Claude Sonnet)
  → 응답 생성 + 출처 인용
```

질문: "이 워크플로우로 진행할까요? 추가할 단계가 있나요?"

## Q5: AI 비용 최적화 전략

"AI 비용을 어떻게 관리할까요?"

- [ ] 캐싱 (같은 질문에 같은 답변 재사용)
- [ ] 모델 계층화 (간단한 질문 → 소형 모델, 복잡한 질문 → 대형 모델)
- [ ] 프롬프트 압축 (긴 컨텍스트 요약)
- [ ] 사용량 제한 (사용자당 일일 한도)
- [ ] 배치 처리 (실시간 불필요한 요청 모아서 처리)

"AI 성능 개선을 위해 데이터를 어떻게 수집할까요?"
