# Scenario 4: 퀀트 투자 — 모멘텀 자동매매 봇

## 난이도: 고급

## 프롬프트

```
바이낸스 BTC/USDT 모멘텀 전략 자동매매 봇을 만들어줘.
- 전략: 20일 이동평균 돌파 시 매수, 이탈 시 매도
- 백테스트: 최근 1년 데이터로 성과 검증
- 리스크 관리: 최대 손실 2%, 포지션 크기 자본의 10%
- 실시간 모니터링: 현재 포지션, 수익률, 신호 상태
- 텔레그램 알림: 매매 신호 발생 시 알림
- dry-run 모드: 실제 주문 없이 시뮬레이션
Python으로, API 키는 .env에서 관리.
```

## 평가 기준 (Assertions)

### 기능 완성도
- [ ] 바이낸스 API에서 과거 캔들 데이터 수집
- [ ] 20일 이동평균 계산이 정확함
- [ ] 돌파/이탈 신호 감지 로직 동작
- [ ] 백테스트 결과 출력 (총수익률, 샤프비율, 최대낙폭)
- [ ] 최대 손실 2% 스탑로스 동작
- [ ] 포지션 사이징 (자본의 10%) 정확
- [ ] dry-run 모드에서 실제 주문 발생 안 함
- [ ] 텔레그램 알림 전송 (또는 시뮬레이션)
- [ ] .env에서 API 키 로드

### 경계면 QA
- [ ] 바이낸스 API 응답 형식 ↔ 내부 캔들 데이터 모델 일치
- [ ] 신호 생성기 출력 ↔ 주문 실행기 입력 일치
- [ ] 백테스트 엔진의 가격 데이터 형식 ↔ 실시간 데이터 형식 일치
- [ ] 수수료/슬리피지 모델이 백테스트와 실거래에서 동일

### 보안 QA
- [ ] API 키가 코드에 하드코딩되어 있지 않음
- [ ] .env 파일이 .gitignore에 포함
- [ ] API 시크릿 키 로그 출력 없음

### 소크라테스 인터뷰 품질
- [ ] "전략 유형" 관련 질문 발생
- [ ] "실행 주기 (초단타/분봉/일봉)" 질문 발생
- [ ] "리스크 허용 범위" 질문 발생
- [ ] "실거래 vs 시뮬레이션 우선" 질문 발생

### 의도적 함정
1. 바이낸스 API rate limit (1200 req/min) → 과거 데이터 대량 수집 시 초과
2. 이동평균 계산에서 첫 19개 캔들은 NaN → 신호 생성기가 NaN 처리 안 하면 크래시
3. 주문 수량의 소수점 자릿수가 바이낸스 심볼별 step_size와 불일치
4. 타임존: 바이낸스 UTC vs 로컬 시간 불일치로 신호 타이밍 오류
