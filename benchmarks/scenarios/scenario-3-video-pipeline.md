# Scenario 3: 영상 제작 — 숏폼 자동 생성 파이프라인

## 난이도: 고급

## 프롬프트

```
유튜브 Shorts용 자동 영상 생성 파이프라인을 만들어줘.
입력: 주제 키워드 (예: "AI 트렌드 2026")
파이프라인:
1. 트렌드 리서치 (웹 검색으로 최신 정보 수집)
2. 대본 생성 (60초 분량, 훅-본문-CTA 구조)
3. TTS 음성 생성 (ElevenLabs API 또는 대안)
4. 자막 파일 생성 (SRT 형식, 타이밍 동기화)
5. 썸네일 텍스트 생성
6. 메타데이터 생성 (제목, 설명, 태그, 해시태그)
Python CLI로 만들고, 각 단계를 독립적으로 실행할 수 있게 해줘.
```

## 평가 기준 (Assertions)

### 기능 완성도
- [ ] 키워드 입력 → 웹 검색 → 정보 수집 동작
- [ ] 대본이 60초 분량 (150~200단어) 생성
- [ ] 대본 구조가 훅(5초)-본문(45초)-CTA(10초)를 따름
- [ ] SRT 자막 형식이 유효함
- [ ] 자막 타이밍이 대본 길이와 대략 일치
- [ ] 메타데이터 JSON 출력
- [ ] 각 단계를 개별 CLI 명령으로 실행 가능
- [ ] 전체 파이프라인 일괄 실행 가능

### 경계면 QA
- [ ] 대본 생성기 출력 형식 ↔ TTS 입력 형식 일치
- [ ] TTS 출력 길이 ↔ 자막 타이밍 동기화
- [ ] 파이프라인 단계 간 파일 경로 참조가 일관됨

### 소크라테스 인터뷰 품질
- [ ] "숏폼 vs 롱폼?" 질문이 발생
- [ ] "TTS 음성 스타일 선호?" 질문이 발생
- [ ] "자동 업로드까지 필요한가?" 범위 질문이 발생

### 의도적 함정
1. TTS API가 비동기 → 결과 폴링 필요한데 동기적으로 호출
2. SRT 타이밍 계산에서 TTS 실제 길이와 예측 길이 불일치
3. 메타데이터의 해시태그 수가 YouTube 제한(30개) 초과
