# Scenario 1: 웹 E-Commerce — 주문 관리 대시보드

## 난이도: 중급

## 프롬프트

```
판매자용 주문 관리 대시보드를 만들어줘.
Next.js + Supabase 기반으로, 다음 기능이 필요해:
- 신규/처리중/완료 주문 목록 조회 (탭 필터)
- 주문 상세 보기 (상품, 금액, 배송지)
- 배송 처리 버튼 (상태 변경: 접수→배송중→완료)
- 일별 매출 요약 차트
반응형이어야 하고, 다크모드 지원해줘.
```

## 평가 기준 (Assertions)

### 기능 완성도
- [ ] 주문 목록 페이지가 존재하고 데이터가 표시됨
- [ ] 탭 필터 (신규/처리중/완료)가 동작함
- [ ] 주문 상세 페이지에서 상품, 금액, 배송지 확인 가능
- [ ] 배송 처리 버튼 클릭 시 상태가 변경됨
- [ ] 상태 변경 후 목록에 즉시 반영됨
- [ ] 매출 요약 차트가 표시됨
- [ ] 반응형 (모바일 375px에서 깨지지 않음)
- [ ] 다크모드 토글 동작

### 경계면 QA
- [ ] API 응답 shape과 프론트 훅 타입이 일치
- [ ] 주문 상태 전이 맵과 실제 업데이트 코드가 일치
- [ ] 모든 href가 실제 존재하는 페이지를 가리킴
- [ ] Supabase 스키마 필드명과 API 응답 필드명 일관성

### Live Browser QA
- [ ] 로그인 → 대시보드 → 주문 클릭 → 배송 처리 유저 저니 성공
- [ ] 유령 사용자가 주요 기능에 접근 가능
- [ ] 파괴자: 빈 주문, 중복 배송 처리 버튼 연타 방어

### 의도적 함정 (Pulse가 발견해야 하는 것)
1. 주문 API가 `{ orders: [...] }`로 감싸서 반환 → 프론트가 배열 직접 기대하는 불일치
2. 배송 처리 후 목록으로 돌아갈 때 route group 접두사 누락 가능성
3. 상태 전이에서 "접수→완료" 직접 전이가 맵에 없지만 코드에서 시도할 수 있음

## 예상 결과

| Live QA 통과율 | 측정 안 됨 | 90%+ |
