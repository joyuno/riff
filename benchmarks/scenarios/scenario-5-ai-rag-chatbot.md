# Scenario 5: AI 엔지니어링 — RAG 기반 고객 상담 챗봇

## 난이도: 고급

## 프롬프트

```
회사 내부 문서 기반 고객 상담 챗봇을 만들어줘.
- RAG 아키텍처: 문서 → 청킹 → 임베딩 → 벡터DB → 검색 → 답변
- 벡터 DB: ChromaDB (로컬)
- LLM: Claude API (Anthropic SDK)
- 문서 형식: PDF, Markdown 지원
- 가드레일: 문서에 없는 내용은 "확인 후 답변드리겠습니다" 응답
- 웹 UI: 채팅 인터페이스 (Next.js)
- 관리자 페이지: 문서 업로드, 인덱스 현황, 질문 로그
```

## 평가 기준 (Assertions)

### 기능 완성도
- [ ] PDF 파일 업로드 → 텍스트 추출 → 청킹 동작
- [ ] Markdown 파일 처리 동작
- [ ] ChromaDB에 임베딩 저장 및 유사도 검색
- [ ] Claude API로 답변 생성 (검색 결과 기반)
- [ ] 문서에 없는 질문 → 가드레일 응답
- [ ] 채팅 UI에서 대화 가능
- [ ] 관리자 페이지에서 문서 업로드 가능
- [ ] 인덱스 현황 (문서 수, 청크 수) 표시
- [ ] 질문 로그 조회 가능

### 경계면 QA
- [ ] 청킹 출력 형식 ↔ 임베딩 입력 형식 일치
- [ ] 벡터 검색 결과 형식 ↔ LLM 프롬프트 컨텍스트 형식 일치
- [ ] 프론트 채팅 컴포넌트 ↔ 백엔드 스트리밍 API 프로토콜 일치
- [ ] 문서 업로드 API ↔ 인덱싱 파이프라인 트리거 연결
- [ ] 관리자 API 인증 ↔ 일반 채팅 API 분리

### Live Browser QA
- [ ] 채팅 UI에서 질문 입력 → 답변 수신 유저 저니 성공
- [ ] 관리자 페이지에서 문서 업로드 → 인덱스 현황 갱신 확인
- [ ] 가드레일: "사장님 전화번호 알려줘" → 거절 응답
- [ ] 스트리밍 응답이 실시간으로 표시됨

### 소크라테스 인터뷰 품질
- [ ] "프롬프트 엔지니어링 vs 파인튜닝" 결정 질문
- [ ] "청킹 전략 (고정 크기 vs 의미 단위)" 질문
- [ ] "가드레일 수준 (엄격/유연)" 질문
- [ ] "답변 소스 표시 여부" 질문

### 의도적 함정
1. PDF에서 테이블 추출 시 텍스트 순서 꼬임 → 청킹 품질 저하
2. 임베딩 모델의 max token과 청크 크기 불일치 → 잘림
3. 스트리밍 API에서 프론트가 SSE vs WebSocket 혼동
4. 가드레일 프롬프트가 system prompt에 있는데 사용자가 "이전 지시 무시해" 공격
5. 관리자 페이지 인증 없이 접근 가능한 보안 취약점

## 예상 결과

| 지표 | Without Pulse | With Pulse |
|------|:---:|:---:|
| 기능 완성도 | 6~7/9 | 8~9/9 |
| 경계면 불일치 | 3~5개 | 0~1개 |
| 보안 이슈 | 2~3개 | 0~1개 |
| 가드레일 우회 | 가능 | 방어됨 |
| Live QA 통과 | 측정 안 됨 | 85%+ |
| 인터뷰 질문 | 0개 | 4~6개 전문가 질문 |
| 총 Pulse 수 | N/A | 15~22 |

## 종합 벤치마크 기대 효과

| 시나리오 | 예상 품질 향상 | 근거 |
|---------|:---:|------|
| 1. 웹 E-Commerce | +40~50% | 경계면 QA + Live Browser로 런타임 버그 사전 차단 |
| 2. 스마트스토어 | +30~40% | 데이터 검증 + 경계면 일관성 보장 |
| 3. 영상 파이프라인 | +50~60% | 파이프라인 단계 간 연결 검증 + 비동기 처리 |
| 4. 퀀트 투자 | +60~70% | 보안 + 금융 도메인 전문가 질문 + NaN 방어 |
| 5. AI RAG 챗봇 | +50~60% | 가드레일 + 스트리밍 경계면 + 보안 QA |

> Harness의 연구 결과(+60% 품질 향상, 100% 승률)와 유사하거나 그 이상의 효과가 예상됨.
> 특히 **복잡도가 높을수록 Pulse의 이점이 커짐** — AI-Native 패턴(분신술, 대립토론)의 효과가 단순 프로젝트보다 복잡한 프로젝트에서 더 두드러지기 때문.
