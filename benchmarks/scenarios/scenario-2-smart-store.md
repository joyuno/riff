# Scenario 2: 스마트스토어 — 상품 자동 등록 시스템

## 난이도: 중급

## 프롬프트

```
네이버 스마트스토어 상품 자동 등록 도구를 만들어줘.
엑셀(xlsx)에서 상품 정보를 읽어서 처리하는 CLI 도구로:
- 엑셀 파싱 (상품명, 가격, 카테고리, 이미지URL, 설명)
- 가격 자동 조정 (원가 대비 마진율 계산, 경쟁가 비교)
- 상품 데이터 검증 (필수 필드 누락, 이미지 URL 유효성)
- 네이버 커머스 API 형식으로 JSON 변환
- 등록 시뮬레이션 (dry-run 모드)
Python으로 만들어줘.
```

## 평가 기준 (Assertions)

### 기능 완성도
- [ ] xlsx 파일 파싱이 동작함
- [ ] 필수 필드 (상품명, 가격) 누락 시 에러 메시지
- [ ] 마진율 계산이 정확함 (원가 × (1 + 마진율) = 판매가)
- [ ] 이미지 URL 유효성 검증 (http/https, 확장자)
- [ ] 네이버 커머스 API 형식의 JSON 출력
- [ ] dry-run 모드에서 실제 API 호출 없이 결과 미리보기
- [ ] 대량 처리 (100개+ 상품) 시 성능 문제 없음

### 경계면 QA
- [ ] 엑셀 컬럼명 ↔ 내부 데이터 모델 필드명 매핑 일관성
- [ ] 내부 모델 ↔ 네이버 API 요청 형식 매핑 일관성
- [ ] 가격 계산 로직의 소수점/반올림 처리 일관성

### 학습 효과
- [ ] 가격 필드에 문자열이 섞인 경우 → 항체 생성
- [ ] 엑셀 헤더가 한글/영문 혼용 → 항체 생성

### 의도적 함정
1. 엑셀의 가격 열에 "1,000" (콤마 포함 문자열) → 숫자 변환 실패
2. 카테고리 코드가 네이버 API의 카테고리 ID와 다른 형식
3. 이미지 URL이 http인데 API는 https만 허용
