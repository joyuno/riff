# Benchmark Fixture: 모호한 자동매매 봇 요청

## Fixture ID
`interview-trading`

## 사용자 프롬프트

> "자동매매 봇 만들어줘"

---

## 평가 목표

퀀트 트레이딩 도메인의 모호한 요청에 대해 Riff가 얼마나 전문적인 인터뷰 질문을
하는지 평가합니다.

이 도메인은 전략 유형·자산·거래소·리스크 관리·규제 등 일반 SW 개발보다 훨씬 많은
의사결정 포인트가 있습니다. 빠뜨리면 실제 자금 손실로 이어지는 질문들이 포함됩니다.

---

## 의도적 모호함 (Embedded Ambiguity)

### STRATEGY — 전략 유형
- 어떤 종류의 전략인지 미명시 (추세추종? 평균회귀? 차익거래? 마켓메이킹?)
- 매매 주기 미명시 (초단타HFT? 일중? 스윙? 포지션?)
- 진입·청산 조건 미명시

### ASSET — 자산 클래스
- 어떤 자산을 거래하는지 불명 (주식? 코인? 선물? 외환?)
- 종목 수 미명시 (단일 종목? 포트폴리오?)
- 국내 vs 해외 미명시

### EXCHANGE — 거래소/브로커
- 어느 거래소를 사용하는지 불명 (업비트, 바이낸스, 한국투자증권, Interactive Brokers...)
- API 방식 미명시 (REST? WebSocket? FIX?)
- 계좌 종류 미명시 (실계좌? 모의투자?)

### RISK — 리스크 관리
- 최대 손실 허용 한도 미명시 (일일? 거래당? 전체?)
- 포지션 사이즈 결정 방식 미명시
- 마진/레버리지 사용 여부 불명

### INFRA — 인프라·운영
- 실행 환경 미명시 (로컬PC? VPS? 클라우드?)
- 24/7 운영 가능 여부 불명
- 레이턴시 요구사항 미명시

### COMPLIANCE — 규제·법적
- 해당 국가·자산의 규제 요건 미확인
- 세금 처리 요구사항 불명
- 알고리즘 트레이딩 라이선스 요건 미확인

```
핵심 질문들을 하세요. 특히 잘못된 결정이 실제 금전적 손실로 이어질 수 있는
퀀트 트레이딩 도메인의 특수성을 고려하세요.
```

## 채점 기준

Ground Truth에는 STRATEGY/ASSET/EXCHANGE/RISK/INFRA/COMPLIANCE 6개 레이어에
총 8개의 필수 질문이 정의되어 있습니다.

- **CRITICAL**: 이 질문 없이는 코드 한 줄도 쓰면 안 됨 (잘못 만들면 돈 잃음)
- **MAJOR**: 아키텍처와 거래소 선택에 직접 영향
- **MINOR**: 운영 품질에 영향

좋은 에이전트 점수: composite ≥ 70%
우수한 에이전트 점수: composite ≥ 85%
