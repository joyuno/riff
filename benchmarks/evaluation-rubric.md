# Pulse 평가 루브릭

## 1. 인터뷰 품질 (Interview Quality) — 25점

### 1-1. 질문 전문성 (10점)

| 점수 | 기준 |
|------|------|
| 9-10 | 도메인 전문가가 묻는 수준의 질문. 트레이드오프를 노출하고, 답변이 아키텍처에 직접 연결됨 |
| 7-8 | 구체적이고 맥락에 맞지만, 일부 질문이 일반적 |
| 5-6 | 기본적인 요구사항 수집. "뭘 만들 건가요?" 수준 |
| 3-4 | 피상적 질문. 답변이 아키텍처 결정에 활용 안 됨 |
| 1-2 | 질문 없이 바로 구현 시작 |

### 1-2. 스코어카드 완성도 (5점)

| 점수 | 기준 |
|------|------|
| 5 | 5 Layer 모두 답변, 모순 감지, 구체성 충분 |
| 4 | 4 Layer 답변, 일부 모순 미감지 |
| 3 | 3 Layer 답변 |
| 2 | 2 Layer 이하 |
| 1 | 스코어카드 없음 |

### 1-3. 답변→아키텍처 매핑 (10점)

| 점수 | 기준 |
|------|------|
| 9-10 | 인터뷰 답변이 기술 스택, 팀 구성, QA 전략에 직접 반영됨 |
| 7-8 | 대부분 반영되나 일부 답변이 무시됨 |
| 5-6 | 부분적으로만 반영 |
| 3-4 | 답변과 아키텍처의 연결이 약함 |
| 1-2 | 인터뷰 결과가 구현에 반영되지 않음 |

## 2. 구현 품질 (Implementation Quality) — 25점

### 2-1. 기능 완성도 (10점)

| 점수 | 기준 |
|------|------|
| 9-10 | MVP 기능 100% 동작, 에러 핸들링 포함 |
| 7-8 | 핵심 기능 동작, 일부 엣지 케이스 미처리 |
| 5-6 | 기본 기능 동작, 중요 기능 일부 누락 |
| 3-4 | 부분적으로만 동작, 주요 기능 누락 |
| 1-2 | 빌드 실패 또는 대부분 미동작 |

### 2-2. 코드 품질 (10점)

| 점수 | 기준 |
|------|------|
| 9-10 | 일관된 구조, 적절한 분리, 타입 안전성, 보안 고려 |
| 7-8 | 대체로 깔끔하지만 일부 개선 여지 |
| 5-6 | 동작하지만 구조적 문제 있음 |
| 3-4 | 하드코딩 다수, 보안 취약 |
| 1-2 | 유지보수 불가능한 수준 |

### 2-3. Pulse 사이클 효율 (5점)

| 점수 | 기준 |
|------|------|
| 5 | 매 Pulse마다 작동하는 결과물, 점진적 진행, 되감기 0회 |
| 4 | 대부분 효율적, 1-2회 비생산적 Pulse |
| 3 | 중간 수준 |
| 2 | 많은 Pulse가 비생산적 |
| 1 | Pulse 구조 없이 직선적 구현 |

## 3. QA 품질 (QA Quality) — 25점

### 3-1. Tier 1 — 경계면 QA (5점)

| 점수 | 기준 |
|------|------|
| 5 | API↔훅, 경로↔링크, 상태전이↔코드 모두 교차 검증 |
| 4 | 3개 영역 중 2개 교차 검증 |
| 3 | 1개 영역만 교차 검증 |
| 2 | 존재 확인만 (교차 비교 없음) |
| 1 | 경계면 검증 없음 |

### 3-2. Tier 2 — 빌드 QA (5점)

| 점수 | 기준 |
|------|------|
| 5 | tsc + eslint + build 모두 통과, 타입 우회 없음 |
| 4 | 빌드 통과, 경미한 경고 |
| 3 | 빌드 통과하지만 타입 우회(as any) 존재 |
| 2 | 빌드 에러 일부 존재 |
| 1 | 빌드 실패 |

### 3-3. Tier 3 — Live Browser QA (10점)

| 점수 | 기준 |
|------|------|
| 9-10 | Playwright로 유저 저니 실행, 스크린샷 증거, 유령+파괴자 QA 포함 |
| 7-8 | 유저 저니 실행 + 스크린샷, 변형 QA 일부 |
| 5-6 | 기본 시나리오만 Playwright로 검증 |
| 3-4 | 수동 확인 또는 부분적 자동화 |
| 1-2 | Live QA 없음 |

### 3-4. 버그 발견율 (5점)

| 점수 | 기준 |
|------|------|
| 5 | 의도적으로 심은 버그 90%+ 발견 |
| 4 | 70~89% 발견 |
| 3 | 50~69% 발견 |
| 2 | 30~49% 발견 |
| 1 | 29% 이하 |
